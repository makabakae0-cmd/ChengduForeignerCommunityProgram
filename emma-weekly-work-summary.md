# Emma 本周工作总结（Events）

## 1. 本周目标

- 完成活动模块第1周与第2周核心开发目标：
  - 活动列表页、活动详情页、报名页骨架
  - 页面跳转与空状态
  - 列表筛选与状态展示
  - 列表/详情统一走 client 拉取数据

## 2. 本周完成内容

### 2.1 页面与交互

- 完成活动列表页 `pages/event/list.vue`
  - 支持筛选：`全部 / 本周 / 即将开始 / 我的`
  - 支持活动状态展示：报名中/进行中/已结束/已取消
  - 支持空状态、加载态、失败态展示
  - 支持点击卡片跳转活动详情

- 完成活动详情页 `pages/event/detail.vue`
  - 完整展示字段：标题、时间、地点、主办、名额、费用、流程、详情
  - 状态标签展示
  - 空状态/异常状态处理
  - 支持从详情页跳转报名页

- 完成活动报名页骨架 `pages/event/signup.vue`
  - 报名信息表单（姓名、电话）
  - 基础校验与提交反馈（MVP骨架）
  - 页面空状态与活动不存在提示

### 2.2 数据请求层

- 新增统一请求封装 `services/http.ts`
- 新增活动统一 client `services/event.client.ts`
  - 列表与详情统一从 client 获取
  - 请求失败自动 fallback 到 mock 数据，保证演示稳定
- 补充 mock 数据 `services/event.mock.ts`

### 2.3 路由与类型

- 更新 `pages.json`，加入活动报名页路由
- 完善活动类型定义 `types/event.ts`

### 2.4 兼容性与构建问题处理

- 修复生命周期兼容问题（detail/signup 改为 `onMounted + getCurrentPages` 取参数）
- 调整前端配置与兼容 shim，解决构建阶段导出不一致问题
- 对齐并修复移动端依赖组合，完成构建链路排障
