# 点位采集审核与映射说明 v1

## 目的

本说明供运营、审核人员、后台录入人员使用，用来把志愿者提交的点位采集卡转换成系统中的 canonical `place` 记录。

核心原则：

- 志愿者提交的是“采集卡”，不是最终发布数据。
- 审核时必须区分哪些字段可直接入库，哪些字段需要规范化，哪些字段只用于复核，哪些字段必须由后台控制。
- 志愿者证据和可信度信息默认不进入公开 `places` payload。

## 四类映射桶

### 1. `direct-map`

志愿者字段经过基础校验后，可直接映射到 canonical `place` 字段。

### 2. `admin-normalize`

志愿者字段有参考价值，但通常需要后台统一润色、翻译、归一化或结构化后才能写入 canonical `place`。

### 3. `review-only`

只用于复核、追溯、判断可信度，不直接成为公开地点字段。

### 4. `admin-owned`

必须由后台或运营掌控，不向志愿者开放填写，也不从志愿者数据直接继承。

## 志愿者字段到 canonical `place` 的映射

| 志愿者字段 | 建议桶位 | 目标字段/去向 | 审核说明 |
| --- | --- | --- | --- |
| 中文名 | `direct-map` | `name_zh` | 以门头、官方名称、长期稳定叫法为准。 |
| 英文名/英文别称 | `admin-normalize` | `name_en` | 有官方英文名可直接用；非官方翻译需复核。 |
| 点位类型 | `direct-map` | `category_level_1` | 必须映射到当前共享 taxonomy。 |
| 志愿者补充子类 | `admin-normalize` | `category_level_2` | 可先记录原词，后台再统一规范。 |
| 别名/旧名/常用叫法 | `review-only` | 审核备注 | 可帮助核对，但不一定进入公开字段。 |
| 详细地址 | `direct-map` | `address_zh` | 优先使用地图标准地址，再按现场信息校对。 |
| 位置凭证 | `review-only` | 审核凭证 | 包括地图截图、定位链接、POI 页面截图等。 |
| 入口说明 | `admin-normalize` | 地址备注 / 导航复核输入 | 当前 `PlaceSchema` 无独立字段，先进入审核说明或后续设计字段。 |
| 路线补充备注 | `review-only` | 审核备注 | 用于确定地图点应落在主入口还是建筑中心。 |
| 主要服务 | `admin-normalize` | `intro_zh` 或标签输入 | 志愿者描述通常较口语化，需要整理后再发布。 |
| 营业/开放时间 | `admin-normalize` | `business_hours_zh` / `business_hours_en` | 中文可先入，英文需补齐或翻译确认。 |
| 联系方式 | `review-only` | 审核备注/后续扩展字段 | 当前 `PlaceSchema` 无联系方式字段。 |
| 适合人群 | `admin-normalize` | `tag_ids` / 推荐文案素材 | 需转成可控标签或文案。 |
| 费用信息 | `review-only` | 审核备注/推荐素材 | 当前无标准价格字段。 |
| 语言支持 | `admin-normalize` | `tag_ids` / 推荐素材 | 建议统一成标签，如英语友好、双语菜单。 |
| 无障碍情况 | `admin-normalize` | `tag_ids` / 推荐素材 | 建议先标签化，不直接写成长文本。 |
| 标签 | `admin-normalize` | `tag_ids` | 必须收敛到系统允许的标签集。 |
| 现场经营/开放状态 | `review-only` | 审核状态备注 | 这不是 `place.status`；`place.status` 表示发布状态。 |
| 信息来源 | `review-only` | 审核凭证 | 用于判断可信度。 |
| 采集时间 | `review-only` | 审核凭证 | 用于判断信息时效性。 |
| 采集人 | `review-only` | 审核凭证 | 用于追溯。 |
| 是否实地到访 | `review-only` | 审核凭证 | 影响复核优先级。 |
| 是否需要复核 | `review-only` | 审核流程输入 | 需要配合复核原因使用。 |
| 复核原因 | `review-only` | 审核流程输入 | 决定是否需要二次确认或补拍。 |
| 门头/入口照片 | `admin-normalize` | `gallery_file_ids` / `gallery_urls` / 封面候选 | 先作为素材，封面选择由后台决定。 |
| 环境照片 | `admin-normalize` | `gallery_file_ids` / `gallery_urls` | 需筛选后进入公开图集。 |
| 补充备注 | `review-only` | 审核备注 | 包括“容易走错”“地图导错”等。 |
| 腾讯地图 POI 链接/ID（如有） | `direct-map` | `tencent_map_poi_id` | 可直接保留，便于后续校对。 |

## 与当前 `place` 结构的术语对齐

当前系统中 `place` 相关核心字段包括：

- `name_zh`
- `name_en`
- `category_level_1`
- `category_level_2`
- `tag_ids`
- `address_zh`
- `address_en`
- `location`
- `tencent_map_poi_id`
- `business_hours_zh`
- `business_hours_en`
- `intro_zh`
- `intro_en`
- `gallery_file_ids`
- `gallery_urls`
- `recommended_reason_zh`
- `recommended_reason_en`
- `is_recommended`
- `recommended_rank`
- `status`

审核与录入时请注意：

- 志愿者说的“现场经营/开放状态”不是 `status`
- `status` 仅表示后台发布状态，如 `draft` / `published`
- 志愿者证据不进入 `location` 以外的公开 payload
- `recommended_rank`、`recommended_reason_*`、`is_recommended` 不应由志愿者决定

## 当前共享 taxonomy 对齐

志愿者类型必须最终落到以下 top-level taxonomy：

| 志愿者标签 | `category_level_1` |
| --- | --- |
| 社区服务 | `public-service` |
| 餐饮 | `food-drink` |
| 购物 | `shopping` |
| 生活服务 | `lifestyle` |
| 教育 | `education` |
| 医疗健康 | `health-wellness` |
| 休闲娱乐 | `entertainment` |
| 户外运动 | `outdoor-sports` |
| 交通 | `transport` |
| 社区/公共空间 | `community` |

`category_level_2` 当前仍是字符串字段，因此可以先记录志愿者提供的子类描述，再由后台统一规范。

## 哪些字段由后台控制

以下字段保留为 `admin-owned`：

- `status`
- `is_recommended`
- `recommended_rank`
- `recommended_reason_zh`
- `recommended_reason_en`
- `supports_navigation`
- `supports_favorite`
- `supports_share`
- `cover_file_id`
- `cover_url`
- 最终对外发布的双语简介文案
- 最终对外发布的英文名、英文地址、英文营业时间

## 发布前审核规则

### 可进入审核池，但不一定可发布

以下情况不阻止收件，但必须进入审核：

- 英文名缺失
- 英文地址缺失
- 英文营业时间缺失
- 主要服务描述较口语化
- 标签为自由文本
- 只有地图平台信息，没有实地到访

### 不满足以下条件时，不应直接发布

- 中文名无法确认
- 一级分类无法确定
- 地址无法确认
- 没有可复核的位置凭证
- 没有可确定的地图点位或入口判断
- 志愿者明确标记“需要复核”且原因尚未关闭
- 关键事实互相冲突，如门头名称与地图名称明显不一致

### 发布前建议补齐的 canonical 字段

在当前实现下，发布前建议确保以下字段已被后台补齐或确认：

- `name_zh`
- `name_en`
- `category_level_1`
- `category_level_2`
- `address_zh`
- `address_en`
- `location`
- `business_hours_zh`
- `business_hours_en`
- `intro_zh`
- `intro_en`
- `tag_ids`
- `status`

## 缺失与不确定信息的处理

### 英文信息缺失

- 可收件
- 不建议直接发布
- 优先查官方英文名，其次再做人工翻译

### 营业时间不确定

- 可收件
- 标记为需复核
- 优先以门头、官方账号、电话确认结果为准

### 联系方式缺失

- 不阻止收件
- 当前不影响 canonical `place` 入库
- 如点位价值高，可单独补采

### 地图地址与现场信息冲突

- 先保留两者
- 必须记录在审核备注中
- 以实地门头和可导航入口为最终判断依据

## 推荐审核流程

1. 初审：判断记录是否完整到可进入审核池
2. 定位校对：确认标准地址、位置凭证、主入口
3. 内容规范化：整理英文、简介、标签、营业时间
4. 媒体筛选：从志愿者照片中选封面和图集素材
5. 发布决策：由后台设置 `status`、推荐相关字段和展示能力
